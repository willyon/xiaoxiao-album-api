## 项目背景

目标：基于 SigLIP 图像编码器训练一个轻量回归头，使其在不加载 NIMA 模型的情况下，输出符合 AVA/NIMA 审美标准的 1–10 分制评分。

## 总体流程总览

1. 采集训练图片并使用现有 `nima_mobilenetv2.onnx` 离线生成初始审美分数（一次性操作）。
2. 对同一批图片运行 SigLIP 模型，抽取并缓存归一化 embedding。
3. 准备训练数据集（train/val/test）并编写数据加载脚本。
4. 训练线性/小型 MLP 回归头并记录验证指标。
5. 评估模型效果（排序相关性、分布稳定性等）。
6. 导出回归头为 ONNX，组合 SigLIP + 头的推理链路。
7. 集成到线上服务并做验证/监控。

## 详细步骤

### 1. 数据准备

- **图片集合**：选取约 6k 张能代表实际业务分布的照片，结构示例：
  - `/data/photos/2024/001.jpg`
  - `/data/photos/...`
- **NIMA 打分脚本**：
  1. 编写离线脚本加载 `nima_mobilenetv2.onnx`（ONNX Runtime）。
  2. 对每张图片执行推理，获得 1–10 分数（浮点）。
  3. 将结果写入 CSV/Parquet，例如：
     ```
     path,score
     /data/photos/2024/001.jpg,7.83
     /data/photos/2024/002.jpg,5.42
     ```
- **一次性操作**：NIMA 仅用于离线生成标签，后续线上不会加载。

### 2. SigLIP embedding 抽取

- 复用现有 SigLIP 推理代码（PyTorch 或 ONNX）。
- 对上述每张图片生成 768 维 embedding，并做 L2 归一化。
- 建议输出为 `.npy` 或 `.pt` 文件，键为图片路径，值为 embedding。
- 若图片量大，可分批存储，并记录哈希校验。

### 3. 构建训练数据集

- 读取 CSV（路径、NIMA 分数）与 embedding 文件。
- 对齐数据：确保每条样本拥有 `embedding` + `score`。
- 划分集合（建议）：
  - 训练集 80%（≈4800）
  - 验证集 10%（≈600）
  - 测试集 10%（≈600）
- 数据加载器：`TensorDataset` + `DataLoader`，批大小 256（可调）。

### 4. 模型与训练

- **模型选择**：
  - 优先尝试线性层：`nn.Linear(768, 1)`。
  - 如需提升，可试 `nn.Sequential(Linear(768,128), GELU, Linear(128,1))`。
- **输出约束**：`score = 1.0 + 9.0 * sigmoid(raw)` 以限制在 1–10。
- **训练配置**：
  - 优化器：`AdamW(lr=1e-3, weight_decay=1e-4)`。
  - Loss：`SmoothL1Loss`。
  - Epoch：10 起步，视验证集表现调整。
  - 记录指标：训练/验证 loss、Spearman/Pearson 相关系数。
- **监控过拟合**：若验证 loss 上升，可降低模型容量或增加正则。

### 5. 评估

- 在验证/测试集上输出：
  - Loss（SmoothL1、MSE）；
  - Spearman 相关系数；
  - 预测分布（均值、方差、分位数）。
- 通过 Top-K 排序结果人工抽检，确认符合主观感受。

### 6. 模型导出

- 训练完成后切换 `head.eval()`。
- 使用标准 `torch.onnx.export` 导出：
  - 输入名：`embedding`（动态 batch）；
  - 输出名：`score`；
  - `opset_version=17`；
  - Dummy 输入：`torch.randn(1, 768)`。
- 确认导出的 ONNX 在 ONNX Runtime 中推理正确。

### 7. 推理链路整合

- 上线时流程：
  1. 上传图片 → SigLIP ONNX → 768 维 embedding。
  2. 将 embedding 输入审美回归头 ONNX → 得到分数。
  3. 分数存入数据库，供排序/展示使用。
- 确保推理服务只加载 SigLIP + 小头，避免加载 NIMA。
- 监控推理耗时与资源占用，验证满足性能要求。

### 8. 验证与回归

- 对比旧流程（SigLIP + NIMA 联合推理）的评分结果，确保新头与 NIMA 一致性在可接受范围。
- 若两者差异较大，检查数据处理、训练设置或考虑增量数据/模型容量。

### 9. 后续迭代方向

- 增量更新：定期用新图片 + NIMA 标签重新训练或微调。
- 模型改进：尝试更深 MLP、加入 Dropout、或对 embedding 做 PCA/特征增强。
- 评估指标自动化：建立每次训练后的评估报告，方便比对历史版本。
