# 环境变量配置模板

## 清理相关配置

### CLEANUP_SIMILAR_HAMMING_THRESHOLD

**相似图检测的汉明距离阈值**

- **类型**: 整数
- **默认值**: `8`
- **说明**:
  - 用于判断两张图片是否相似（基于 pHash 感知哈希的汉明距离）
  - 汉明距离越小，图片越相似
  - 相似度分数计算公式：`similarity = 1 - (hamming_distance / 64)`
- **阈值建议**:
  - `≤ 5`: 非常严格，只识别几乎完全相同的图片（精确率高，召回率低）
  - `8`: 平衡阈值（推荐，精确率和召回率平衡）
  - `≥ 10`: 宽松阈值，可能包含更多相似图（精确率低，召回率高）

- **示例**:

  ```
  CLEANUP_SIMILAR_HAMMING_THRESHOLD=8
  ```

- **算法说明**:
  - 使用 pHash（感知哈希）算法，基于 DCT（离散余弦变换）
  - 生成 64 位二进制哈希，转换为 16 位十六进制字符串
  - 计算两个哈希值的汉明距离（不同位的数量）
  - 汉明距离 ≤ 阈值时，判定为相似图

- **性能优化**:
  - 使用倒排索引优化，按哈希前缀分组
  - 时间复杂度：从 O(n²) 优化到 O(n × k)
  - 支持前缀剪枝，大幅减少不必要的比较

### BLURRY_SHARPNESS_THRESHOLD

**模糊图检测的清晰度阈值**

- **类型**: 浮点数
- **默认值**: `0.25`
- **说明**:
  - 清晰度分数 < 此值时，判定为模糊图
  - 清晰度分数范围：0~1（值越大越清晰）
  - 判断规则：`sharpness_score < 阈值 → 判定为模糊图`

- **阈值建议**:
  - `0.2`: 严格阈值，只识别严重模糊的图片（精确率高，召回率低）
  - `0.25`: 平衡阈值（推荐，精确率和召回率平衡）
  - `0.3`: 宽松阈值，可能误判一些清晰度较低的图片（精确率低，召回率高）
  - `0.35`: 非常宽松，适合需要高召回率的场景

- **示例**:

  ```
  BLURRY_SHARPNESS_THRESHOLD=0.25
  ```

- **算法说明**:
  - 使用传统清晰度检测方法（多指标融合）
  - 包括：Laplacian variance、Tenengrad、Brenner gradient、SMD
  - 边缘区域检测（只计算边缘区域的清晰度，更准确）
  - 对数归一化（处理大动态范围）
  - 轻微高斯去噪（减少噪声干扰）
